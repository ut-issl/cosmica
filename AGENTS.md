## Project Overview

COSMICA (COnstellation Satellite siMulator for optIcal CommunicAtion) is a typed Python 3.13+ library for modeling and simulating optical communication satellite constellations. Its composable modules model space and ground nodes, propagate orbital and solar dynamics, construct time-varying NetworkX topologies, calculate directional link availability and performance, generate geographic scenarios, and visualize network snapshots. Numerical code uses NumPy arrays and SI units.

Dependencies should flow toward the lower layers. The project uses `uv` for dependency and environment management, `uv_build` for packaging, pytest and Coverage.py for tests, Ruff for linting and formatting, multiple static type checkers, and MkDocs for documentation.

## Repository Layout

- `src/cosmica/`: installable package in a `src` layout.
  - `models/`: domain models for constellations, satellites, orbits, terminals, gateways, users, demands, and scenarios.
  - `dtos/`: validated data transferred between simulation stages, principally time-series dynamics data.
  - `dynamics/`: orbit propagation and Sun-direction calculations.
  - `topology/`: NetworkX graph builders for inter-satellite, satellite-to-ground, gateway, and user-satellite links.
  - `comm_link/`: directional communication-link calculators and their coordinator; computes availability, capacity, and delay.
  - `scenario/`: population, maritime, and gateway-network scenario data and generators.
  - `visualization/`: plotting and equirectangular map utilities, plus bundled map assets.
  - `utils/`: shared constants, coordinate transforms, Gaussian-beam calculations, and vector operations.
- `tests/`: pytest suite organized by package area; orbit snapshot tests use Syrupy snapshots.
- `examples/`: runnable basic simulation and Marimo orbit-propagation examples.
- `docs/`: hand-written MkDocs pages and documentation assets; API pages are generated by `scripts/gen_ref_pages.py`.
- `.github/`: CI, documentation, release, security workflows, and issue/PR templates.
- `pyproject.toml`: package metadata, dependency groups, tool configuration, and the import-linter architecture contract.
- `uv.lock`: reproducible dependency lockfile.
- `justfile`: common lint, type-check, test, license, and documentation commands.

## GitHub Issues and Pull Requests

When opening or updating a GitHub Pull Request, comply with `.github/pull_request_template.md`.

When an agent files a GitHub Issue or opens a GitHub Pull Request, put this alert note at the very top of the description. Add the same note at the beginning of any issue or pull request comment written by an agent:

> [!WARNING]
> This content was written by an AI agent and must be verified by a human developer. After human verification, this alert may be removed.

The human developer must remove this note after verifying the description or comment's contents.
